# Phase 3 Auth (Clerk invite-only access control) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Gate the page and `GET /api/readiness` behind an invite-only Clerk allowlist (email magic-link) so only invited `@viewnear.com` / `@lendzfinancial.com` users see the live delivery data.

**Architecture:** Approach A — the SPA renders a Clerk sign-in screen until authenticated (`@clerk/clerk-react`); the authed client sends the Clerk session token on every `/api/readiness` call; the function verifies it server-side with `@clerk/backend` and returns 401 without a valid token. The cron path (`/api/refresh`, `CRON_SECRET`) and the `ReadinessPayload` shape are unchanged.

**Tech Stack:** TypeScript, Vite + React 19, `@vercel/node` functions, `@clerk/clerk-react`, `@clerk/backend`, `@vercel/blob`, Vitest. Spec: `docs/superpowers/specs/2026-06-18-phase3-auth-design.md`.

## Global Constraints

- Language: code, identifiers, comments, commit messages in **English**. No code comments unless they explain a non-obvious *why* (one line max).
- **Do not change the `ReadinessPayload` shape or any per-module field.** This phase only adds an auth layer.
- **ESM extensions:** every relative import inside `api/` MUST carry an explicit `.js` extension (enforced by `api/import-extensions.test.ts`). Test files are exempt and import without `.js`.
- `CLERK_SECRET_KEY` is **server-side only** — never a `VITE_*` var, never committed. `VITE_CLERK_PUBLISHABLE_KEY` is the Clerk *publishable* key (public by design); the `VITE_` prefix is correct and does not violate the no-secrets rule.
- The gated `/api/readiness` response MUST use `Cache-Control: private, no-store` — never a shared/CDN cache (`s-maxage`), which would let the edge serve an authed response to an anonymous request and bypass the gate.
- Tests must never deploy (`.vercelignore` excludes `*.test.ts` / `*.test.tsx`).
- TDD: red → green → refactor. Commit after each green task. Work only on branch `feature/phase3-auth`; never commit to `develop`/`main`.
- Run the full suite with `npx vitest run`; a single file with `npx vitest run <path>`.

```ts
import { verifyToken } from '@clerk/backend'
import type { VercelRequest } from '@vercel/node'

export function getClerkSecret(): string {
  const s = process.env.CLERK_SECRET_KEY
  if (!s) throw new Error('CLERK_SECRET_KEY is not set')
  return s
}

type Claims = Awaited<ReturnType<typeof verifyToken>>

export async function verifyRequest(req: VercelRequest): Promise<Claims | null> {
  const header = req.headers.authorization
  if (!header?.startsWith('Bearer ')) return null
  const secretKey = getClerkSecret()
  try {
    return await verifyToken(header.slice('Bearer '.length), { secretKey })
  } catch {
    return null
  }
}
```

```ts
import type { VercelRequest, VercelResponse } from '@vercel/node'
import { buildPayload } from '../shared/readiness.js'
import { readLatest } from './_lib/blob.js'
import { verifyRequest } from './_lib/auth.js'

export default async function handler(req: VercelRequest, res: VercelResponse) {
  const claims = await verifyRequest(req)
  if (!claims) return res.status(401).json({ error: 'unauthorized' })
  res.setHeader('Cache-Control', 'private, no-store')
  const latest = await readLatest()
  res.status(200).json(latest ?? buildPayload(new Date().toISOString()))
}
```

```ts
import type { ReadinessPayload } from '../shared/readiness'

export async function fetchReadiness(
  getToken: () => Promise<string | null>,
  signal?: AbortSignal,
): Promise<ReadinessPayload> {
  const token = await getToken()
  const res = await fetch('/api/readiness', {
    signal,
    headers: token ? { Authorization: `Bearer ${token}` } : {},
  })
  if (!res.ok) {
    throw new Error(`Failed to load readiness data (${res.status})`)
  }
  return (await res.json()) as ReadinessPayload
}
```

```ts
import { head, put } from '@vercel/blob'
import type { ReadinessPayload } from '../../shared/readiness.js'

const BLOB_PATH = 'readiness/latest.json'

export async function writeLatest(payload: ReadinessPayload): Promise<void> {
  await put(BLOB_PATH, JSON.stringify(payload), {
    access: 'private',
    contentType: 'application/json',
    addRandomSuffix: false,
    allowOverwrite: true,
  })
}

export async function readLatest(fetchImpl: typeof fetch = fetch): Promise<ReadinessPayload | null> {
  try {
    const meta = await head(BLOB_PATH)
    const res = await fetchImpl(meta.url, {
      cache: 'no-store',
      headers: { Authorization: `Bearer ${process.env.BLOB_READ_WRITE_TOKEN}` },
    })
    if (!res.ok) return null
    return (await res.json()) as ReadinessPayload
  } catch {
    return null
  }
}
```
